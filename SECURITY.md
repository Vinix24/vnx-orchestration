# Security Policy

## Reporting a vulnerability

Please report security issues privately to the maintainer at **vincentvd@gmail.com** (or via GitHub's private security-advisory flow on the repository). Do not open a public issue for a security report.

Include:

- clear description of the issue
- impact assessment
- reproduction steps or proof of concept
- suggested remediation if available

Do not open public issues for unpatched vulnerabilities.

## Response expectations

We will triage reports and respond as quickly as practical. Timelines are best-effort and depend on maintainer availability.

## Scope

This policy covers code and templates shipped as part of VNX dist.
